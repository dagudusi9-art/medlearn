# MedLearn — 医療学習プラットフォーム

Atomic Knowledge Card とクイズによる反復学習アプリです。GitHub Pagesだけでホスティングでき、PWA（オフライン対応・ホーム画面追加）に対応しています。

## 構成

```
index.html              アプリの入口（コンテンツは一切含みません）
manifest.webmanifest    PWAマニフェスト
service-worker.js       オフラインキャッシュ
/app
  app.css                デザイン（全教材共通）
  app.js                 アプリ本体のロジック（全教材共通・教材の中身は一切知りません）
/icons                  PWAアイコン
/courses
  courses.json           教材の一覧（IDを追加するだけで新しい教材が表示されます）
  /tavi
    course.json           教材メタデータ（タイトル・説明・章構成など）
    atomic_cards.json      一問一答カード
    quiz_questions.json    四択・○×・穴埋め問題
  /_template              新しい教材を追加する際のひな形（courses.jsonには未登録）
```

**アプリ本体（`index.html` / `app/`）は教材の内容を一切持ちません。** すべてのコンテンツは `/courses/<教材ID>/` 以下のJSONファイルとして分離されています。新しい教材（CABG, AVR, EVAR, MitraClip, ABSITEなど）を追加する方法は `courses/_template/README.md` を参照してください。

## GitHub Pagesへのデプロイ

1. このフォルダの中身をリポジトリのルート（またはルートの `/docs` フォルダ）にそのまま置く
2. GitHubにpush
3. リポジトリの Settings → Pages で、公開元をそのブランチ／フォルダに設定
4. 数分後に `https://<ユーザー名>.github.io/<リポジトリ名>/` でアクセス可能になります

**リポジトリ直下（ユーザーページ）でもプロジェクトページ（サブパス）でも、コード変更なしでそのまま動作します。** すべてのパスは相対パス／現在のURLからの絶対パスで解決される設計になっています。

## ローカルでの動作確認

PWA（Service Worker・fetch）はブラウザの仕様上 `file://` では動作しないため、簡易サーバーを立てて確認してください。

```bash
# このフォルダで実行
python3 -m http.server 8000
# または
npx serve .
```

その後 `http://localhost:8000/` を開いてください。

## PWA対応の内容

- **ホーム画面へ追加**: `manifest.webmanifest` によりスマホ・PCともに「ホーム画面に追加」「インストール」が可能です。インストール後はブラウザのUIなしで通常のアプリのように起動します（`display: "standalone"`）。
- **オフライン動作**: `service-worker.js` がアプリ本体（HTML/CSS/JS/アイコン）をインストール時にキャッシュし、教材データ（`/courses/**`）は初回アクセス時にキャッシュされます。一度開いた教材はオフラインでも学習を継続できます。
- **学習履歴**: これまで通りLocalStorageに保存されます。教材ごとに独立したキー（`medlearn_progress_v1__<教材ID>`）で保存されるため、教材を追加・削除しても他の教材の履歴に影響しません。

### 新しいデプロイを反映させたいとき

`service-worker.js` 冒頭の `CACHE_VERSION` の値を変更してください（例: `medlearn-v1` → `medlearn-v2`）。これにより古いキャッシュが破棄され、次回起動時に新しいアプリ本体が取得されます（教材データ自体は起動のたびにバックグラウンドで最新版を取りに行き、次回表示時から反映されます）。

## 共通学習機能

すべての教材で以下のモードが共通利用できます（教材固有のコードは不要です）。

- 一問一答（Atomic Knowledge Card）
- 四択 / ○× / 穴埋め
- クイズ総合（ミックス出題）
- 間違えた問題だけ復習
- ブックマーク復習
- 重要度フィルタ（★の数）による絞り込み
- Spaced Repetition（正解後 1日→3日→7日後に再出題、誤答は同一セッション内で再出題）
- Learn Mode（教材が指定する章順に、カード学習→章末クイズ→合格して次の章へ、と進む学習パス）

## 今後の拡張について

現在対応しているコンテンツ形式は「Atomic Knowledge Card」と「クイズ問題」の2種類です。将来、動画・画像・PDF・症例ベース問題・チェックリストなどを追加する場合は、`course.json` に新しいコンテンツ種別のフィールド（例: `casesFile`）を追加し、`app/app.js` に対応する表示ロジックを追加する形で拡張してください。既存教材のデータには影響しません。
